# AI 开源仓库中文榜单站

> 精选 GitHub 上 **500+ AI 开源项目**，每个项目都有 LLM 生成的**中文解读**：一句话简介、典型使用场景、上手难度评估、同类替代方案。让你 30 秒看懂一个项目值不值得花时间。

🔗 **在线站点**：https://nanShi-P.github.io/SITE_URL

---

## 站点你能看到什么

### 📊 500 个项目，9 大分类
按方向归类：**LLM · Agent · RAG · 多模态 · 推理与训练 · 工具与基建 · 应用 · 数据 · 其他**。每个分类页按 Star 数排序，一眼看清当前赛道的头部项目。

### 🇨🇳 每个项目都有中文解读
不是简单翻译 README，而是 LLM 在读完 README 后输出的结构化解读：

- **一句话简介** — 用最简洁的中文说清这是什么
- **适用场景** — 3 个典型使用场景，帮你判断"我用得上吗"
- **上手难度** — 简单 / 中等 / 困难，附理由（依赖、文档质量、社区活跃度）
- **同类替代** — 列出可类比的项目，便于横向比较

### 📈 Star 走势与排名变化
每次数据更新都会记录排名变化（`rank_delta`），可以看到哪些项目最近窜升、哪些在掉。

### 🔍 适合谁
- 想跟进 AI 圈但被英文 README 劝退的开发者
- 选型时希望快速横向对比的工程师
- 想了解某个方向头部项目都有谁的研究者 / 学生
- 写技术周报 / 选题时需要灵感的内容创作者

## 数据来源与质量说明

- **数据源**：GitHub Search API，按 AI 相关 topic 拉取并去重
- **更新频率**：每周手动更新一次
- **中文解读**：由 LLM 基于项目 README 生成，已做缓存（README 不变不重复生成）
- ⚠️ AI 生成内容可能存在少量不准确，详情页底部有声明；选型决策仍建议查看原仓库

## 访问统计

站点接入 GoatCounter（无 Cookie）。本地开发可留空：

- 本地：`.env` 中 `PUBLIC_GOATCOUNTER_CODE` 留空 → 不注入脚本
- 部署：在 GitHub 仓库 **Settings → Secrets and variables → Actions → Variables**
  新建 `PUBLIC_GOATCOUNTER_CODE`，值为 GoatCounter 子域前缀（如 `nanship`）
